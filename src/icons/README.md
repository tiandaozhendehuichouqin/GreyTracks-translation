# 图标文件说明

## 当前状态

已生成 SVG 格式的占位图标：
- ✅ icon16.svg
- ✅ icon32.svg
- ✅ icon48.svg
- ✅ icon128.svg

这些图标已经可以在 Chrome 和 Firefox 中正常使用。

## 图标设计建议

### 方案 1：使用现有 SVG 图标（推荐）
当前生成的 SVG 图标包含：
- 紫色渐变背景（#667eea 到 #764ba2）
- 白色"译"字
- 简洁现代的设计风格

### 方案 2：自定义设计
如需自定义图标，请遵循以下规范：

**尺寸要求：**
- 16x16 像素
- 32x32 像素
- 48x48 像素
- 128x128 像素

**格式要求：**
- PNG 格式（推荐）
- 或 SVG 格式（已支持）
- 背景透明
- 色彩鲜明

**设计建议：**
- 使用与翻译相关的图标（书本、对话气泡、翻译符号等）
- 保持简洁，避免过多细节
- 确保在小尺寸下依然清晰可辨
- 主色调建议与插件 UI 一致（紫色渐变）

### 方案 3：在线生成工具

可以使用以下在线工具生成图标：

1. **Favicon Generator**
   - 访问：https://www.favicon-generator.org/
   - 上传设计好的图片
   - 下载多尺寸图标包

2. **Canva**
   - 访问：https://www.canva.com/
   - 搜索"app icon"模板
   - 自定义设计并导出

3. **Figma**
   - 访问：https://www.figma.com/
   - 使用免费模板
   - 导出为 PNG 或 SVG

## 替换图标

如果要替换为自定义图标：

1. 将新图标文件放入 `src/icons/` 目录
2. 确保文件名为：
   - `icon16.png`（或 `.svg`）
   - `icon32.png`（或 `.svg`）
   - `icon48.png`（或 `.svg`）
   - `icon128.png`（或 `.svg`）
3. 重新运行构建：`npm run build`
4. 在浏览器中重新加载扩展

## 注意事项

- Chrome 和 Firefox 都支持 SVG 格式的图标
- 如果使用 PNG 格式，确保为真彩色（24 位或 32 位带 alpha 通道）
- 图标文件应该放在 `src/icons/` 目录，构建时会自动复制到 `dist/` 目录
- 修改图标后需要重新构建并重新加载扩展
