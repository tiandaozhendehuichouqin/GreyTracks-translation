# 跨浏览器划词翻译插件项目计划书

## 1. 项目概要

### 1.1 项目背景
随着互联网的全球化，用户在浏览外文网页时经常遇到语言障碍。浏览器插件作为一种轻量级扩展工具，能够在不离开当前页面的情况下提供即时翻译服务，极大提升用户体验。目前市场上虽有多种翻译插件，但存在跨浏览器兼容性差、功能冗余或隐私顾虑等问题。本项目旨在开发一款简洁、高效、支持 Chrome 和 Firefox 的划词翻译插件，满足用户日常英文阅读需求。

### 1.2 项目目标
- 开发一款浏览器插件，支持在 Chrome（Manifest V3）和 Firefox（Manifest V2/V3）上运行。
- 核心功能：用户选中网页中的英文文本后，自动显示翻译结果（可设置为中文或其他目标语言）。
- 提供简洁的设置界面，允许用户选择目标语言、开启/关闭插件、管理翻译服务。
- 保证插件的轻量化、响应迅速，并尊重用户隐私（尽可能减少数据收集）。

### 1.3 项目范围
**包含：**
- 插件的前端界面（弹出页、设置页、翻译结果弹窗）。
- 后台脚本（Service Worker / Background Page）处理翻译 API 调用。
- 内容脚本（Content Script）实现划词监听和结果展示。
- 跨浏览器适配（使用 polyfill 和两套 manifest 配置）。
- 集成至少一种免费翻译 API（如 LibreTranslate 或百度翻译 API）。
- 基本缓存机制，减少重复翻译请求。

**不包含：**
- 整页翻译功能（初期不实现，可后续迭代）。
- 复杂的用户账户系统。
- 本地机器学习翻译模型。

## 2. 项目需求

### 2.1 功能需求

#### 2.1.1 划词翻译
- 用户在任意网页选中文本（鼠标松开）后，自动或通过快捷键触发翻译。
- 在选中文本附近显示一个精致的翻译结果卡片，包含原文和译文。
- 卡片可拖动或点击关闭，并支持自动消失（可配置延迟）。
- 支持排除特定元素（如输入框、密码框）内的划词。

#### 2.1.2 翻译服务
- 支持切换翻译引擎（例如：谷歌翻译、百度翻译、LibreTranslate 等），至少内置一种可用服务。
- 用户可自定义 API 密钥（针对需要付费或注册的服务）。
- 翻译失败时有友好的错误提示。

#### 2.1.3 设置选项
- 目标语言选择（如中文、英文、日文等，默认中文）。
- 开关插件是否启用。
- 是否显示翻译按钮（hover 按钮再触发翻译，减少自动弹窗干扰）。
- 清除缓存数据。
- 关于页面（版本、开源协议等）。

#### 2.1.4 右键菜单
- 在选中的文本上右键，增加“翻译”选项，点击后调用翻译。

#### 2.1.5 缓存机制
- 本地缓存已翻译的文本（原文-译文对），避免重复请求 API。
- 缓存可设置过期时间或由用户手动清除。

#### 2.1.6 隐私保护
- 不收集用户个人数据。
- 若使用第三方 API，需明确告知用户并遵守其隐私政策。

### 2.2 非功能需求

#### 2.2.1 性能
- 插件启动时间不超过 1 秒。
- 翻译结果弹窗响应迅速，无卡顿。
- 内存占用小，不影响浏览器整体性能。

#### 2.2.2 兼容性
- **Chrome**: 版本 88 及以上（支持 Manifest V3）。
- **Firefox**: 版本 78 及以上（支持 Manifest V2，并逐步兼容 V3）。
- 兼容常见网站（无样式冲突，可在 iframe 内工作）。

#### 2.2.3 易用性
- 用户界面简洁直观，无需学习即可上手。
- 提供详细的使用说明（README 或内置帮助）。

#### 2.2.4 可维护性
- 代码结构清晰，模块化分离（内容脚本、后台、弹出页）。
- 使用 ESLint 统一代码风格。
- 提供完善的注释和文档。

#### 2.2.5 安全性
- 防止 XSS 攻击（对翻译结果进行转义再插入 DOM）。
- 仅请求必要的权限（如 `<all_urls>` 或 `activeTab`）。
- API 密钥安全存储（使用 `chrome.storage.local` 加密存储，建议用户自填）。

### 2.3 跨浏览器特定需求
- 在 Firefox 中使用 `browser` 命名空间，通过 `webextension-polyfill` 统一 API。
- 准备两套 `manifest.json` 分别对应 Chrome 和 Firefox，打包时自动选择。
- 测试在 Chrome 和 Firefox 上的表现一致。

### 3.2 迭代计划

#### 版本 1.0 (MVP)
- 实现核心划词翻译功能（使用预设翻译 API，如 LibreTranslate 免费实例）。
- 支持 Chrome 和 Firefox 基本运行。
- 提供简单的弹出页开关和目标语言选择。
- 完成基础缓存功能（不设过期时间，仅内存缓存）。

#### 版本 1.1
- 增加右键菜单翻译。
- 增加翻译引擎切换（支持百度翻译、谷歌翻译，需用户填写密钥）。
- 优化弹窗样式和动画。
- 增加清除缓存按钮。

#### 版本 1.2
- 支持在 iframe 内工作（如网页内嵌的文档）。
- 增加排除网站列表（用户可设置某些网站不启用插件）。
- 实现设置页的国际化（多语言界面）。

#### 版本 1.3 (可选)
- 整页翻译功能（调用 API 翻译整个可见文本）。
- 文本朗读功能（TTS）。

### 3.3 技术选型

| 技术领域       | 选择方案                                                                 |
|----------------|--------------------------------------------------------------------------|
| 核心语言       | JavaScript (ES6+)                                                        |
| 跨浏览器兼容   | `webextension-polyfill` 库                                               |
| UI 样式        | 原生 CSS（或考虑轻量级 UI 库，如 Shoelace）                              |
| 构建工具       | 可选：Webpack 或 Vite（便于打包两套 manifest 和资源优化）               |
| 翻译 API       | 初期：LibreTranslate 公共实例；后续集成百度翻译、谷歌翻译（需 API key） |
| 存储           | `chrome.storage.local`（异步，容量大）                                   |
| 代码规范       | ESLint + Prettier                                                        |

### 3.4 开发环境与测试
- **开发环境**：Node.js (v16+)，npm/yarn。
- **调试**：Chrome 扩展管理页面加载未打包扩展；Firefox 使用 `about:debugging` 临时加载。
- **自动化测试**：暂不引入，但可编写单元测试（如 API 调用模块）使用 Jest。


## 4. 风险评估与应对

| 风险                                     | 可能性 | 影响 | 应对措施                                                         |
|------------------------------------------|--------|------|------------------------------------------------------------------|
| 翻译 API 失效或限流                       | 中     | 高   | 内置多个备用 API；允许用户自定义 API 端点；增加重试机制。        |
| 浏览器新版本导致 API 变更                 | 低     | 中   | 关注 Chrome 和 Firefox 扩展更新公告，及时适配。                  |
| 插件被恶意网站利用 XSS                     | 低     | 高   | 严格转义输出内容；使用 DOMPurify 净化 HTML；限制内容脚本权限。   |
| 用户反馈影响现有网站布局                   | 中     | 中   | 仔细设计翻译卡片样式，避免覆盖页面重要元素；提供禁用特定网站的选项。 |
| 跨浏览器差异导致部分功能不可用             | 中     | 高   | 广泛使用 polyfill；编写抽象层隔离差异；在两种浏览器上持续测试。   |
